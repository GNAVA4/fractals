# Фракталы — Liquid Glass

Десктопное PyQt6-приложение для визуализации фракталов с прогрессивной анимацией,
8 цветовыми палитрами и стеклянным UI. Ядро написано в функциональном стиле:
чистые функции + frozen-датаклассы, без побочных эффектов.

## Возможности

- **6 типов фракталов**: Серпинский, T-фрактал, H-фрактал, Кох, дерево Пифагора, молния
- **8 палитр** с интерполяцией стопов: Ocean, Aurora, Cosmic, Sunset, Fire, Forest, Neon, Mono — плюс полный HSL Rainbow
- **Прогрессивная анимация** — построение фрактала по уровням рекурсии (BFS) или чанками,
  с сохранением идентичности цветов между анимацией и финальным рендером
- **Симметрии** как алгебраическая группа: отражение по X/Y + n-кратная радиальная.
  Корректная композиция: `reflect_x ∘ reflect_y` = D2 (4 копии), `radial(N) ∘ reflect` = D2N
- **Контекстная видимость параметров** — «угол ветвей» появляется только для дерева,
  seed/шероховатость — только для молнии, и т.д.
- **Навигация холста**: колесо — зум к курсору, drag — панорама, double-click — сброс
- **Масштаб-инвариантная толщина линий** при сильном зуме

## Установка

Требования: **Python 3.10+** (использует `match`-statements, `Literal`, `slots=True`).

### Шаг 1. Клонировать репозиторий

```bash
git clone https://github.com/GNAVA4/fractals.git
cd fractals
```

### Шаг 2. Создать и активировать виртуальное окружение

**Windows (PowerShell / cmd):**
```powershell
python -m venv venv
venv\Scripts\activate
```

**Linux / macOS:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### Шаг 3. Установить зависимости

```bash
pip install -r requirements.txt
```

Единственная runtime-зависимость — `PyQt6` (>= 6.5.0).

## Запуск

```bash
python app.py
```

Откроется главное окно. По умолчанию строится Серпинский с палитрой Aurora.
Параметры (тип, глубина, симметрии, палитра) переключаются на панели слева;
изменения применяются автоматически с дебаунсингом 150 мс.

**Горячие действия в холсте:**

| Действие | Эффект |
|---|---|
| Колесо мыши | Зум к курсору |
| Перетаскивание ЛКМ | Панорама |
| Двойной клик | Сброс зума и панорамы |
| Кнопка «▶ Запуск» | Прогрессивная анимация по уровням |

## Возможные проблемы

| Ошибка | Решение |
|---|---|
| `ModuleNotFoundError: No module named 'PyQt6'` | Активируйте venv и установите зависимости: `pip install -r requirements.txt` |
| `qt.qpa.plugin: Could not load the Qt platform plugin` на Linux | Установите системный пакет: `sudo apt install libxcb-cursor0 libxkbcommon-x11-0` |
| Окно не открывается, нет вывода | Запустите без подавления stdout: `python -u app.py` |

## Архитектура

```
app.py                       # Точка входа (QApplication → MainWindow)
src/
├── core/                    # Чистая логика, без GUI
│   ├── types.py             # Point, Line, FractalConfig (frozen dataclasses)
│   ├── generators.py        # 6 фракталов: lines/batches с гарантированной идентичностью порядка
│   ├── lightning.py         # Молния с детерминированным шумом
│   ├── coloring.py          # 8 палитр + индексное семплирование
│   ├── transforms.py        # translate / rotate / scale (чистые функции)
│   ├── symmetry.py          # reflect_x/y, radial_symmetry; apply_symmetries как orbit-композиция
│   └── utils.py             # compose, apply_to_lines, flatten_lines
├── bridge.py                # Мост core ↔ GUI: generate_fractal / generate_fractal_batched
└── gui/
    ├── main_window.py       # GlassWindow + MainWindow (таймеры, цикл анимации)
    ├── canvas.py            # FractalCanvas: кэш фона в QPixmap, группировка линий по цвету
    └── controls.py          # ControlsPanel + glass-виджеты (свотчи палитр, иконки фракталов)
```

### Ключевые архитектурные решения

| Принцип | Реализация |
|---|---|
| **Ядро без GUI** | `src/core/` не импортирует PyQt; легко покрывается unit-тестами |
| **Идентичность batches и lines** | `*_lines = chain.from_iterable(*_batches(...))` для рекурсивных по уровням; обратно — для регулярных. Гарантирует одинаковый порядок и совпадение цветов |
| **Композиция симметрий** | `apply_symmetries` накапливает орбиту: для каждой sym `result += [sym(l) for l in result]` |
| **Согласованность цветов в анимации** | `ColorParams(total, index_offset)` пробрасывается в каждый батч — индексы продолжаются непрерывно |
| **Перформанс рендера** | Кэш фона в `QPixmap` (резайз-инвалидация), `defaultdict` группировка линий по hex → один `drawLines` на цвет |

## Палитры

Все палитры — хендкрафтнутые `(R,G,B)`-стопы, интерполируются кусочно-линейно. Подобраны так,
чтобы быть читаемыми на тёмном фоне (нет совсем тёмных стопов) и иметь luminance-вариацию для
визуальной глубины.

```python
PALETTES = {
    "ocean":   [(40,70,160), (60,140,220), (90,220,235), (220,250,250)],
    "fire":    [(120,25,60), (220,60,50),  (255,145,50), (255,230,130)],
    "forest":  [(30,110,70), (90,200,100), (190,230,110), (240,250,200)],
    "cosmic":  [(70,20,130), (160,60,220), (235,110,235), (250,210,250)],
    "aurora":  [(40,200,140),(70,190,235), (160,140,245),(225,195,250)],
    "sunset":  [(90,35,130), (220,75,130), (255,145,85), (255,225,135)],
    "neon":    [(50,230,245),(150,90,245), (245,90,210), (255,230,230)],
    "mono":    [(170,195,235),(245,250,255)],
}
```
